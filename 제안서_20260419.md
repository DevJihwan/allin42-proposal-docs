## 1. 배경 및 목표

### v1 대비 주요 변경점 (요약)

| 항목 | v1 (설치형) | v2 (관리형 SaaS) |
|------|------------|-----------------|

| 사용 형태 | 코드 설치·터미널 실행 | **웹 대시보드 (URL 접속)** |
| 접근성 | 특정 PC에서만 | **어디서든·모든 기기** |

| 다계정 관리 | 스크립트 수동 수정 | **10개 계정 병렬 자동화** |
| A/S | 원격 접속 필요 | **대시보드 로그 공유만으로 대응** |

> 상세 기능·확장성 비교는 **§16 v1 vs v2 전체 비교표** 참조

### 구현 목표

- **Cloudflare 기반 24/7 자동 운영**: PC 의존 완전 제거
- **웹 대시보드 (클라이언트용 + 관리자용 2종)**: 실시간 제어·모니터링
- **10개 계정 병렬 운영**: 사이트당 작업량 10배 증대
- **자동 로그인**: 세션 감지 즉시 재로그인 (공백 시간 제거)
- **관리형 서비스**: 개발자가 인프라 운영, 클라이언트는 대시보드만 사용

### 기대 효과

- 댓글 작업 시간 **100% 자동화**
- PC·사람 개입 없이 24시간 무중단 운영
- 10개 계정 병렬로 **일일 작업량 최대 10배 증가**
- 원격 모니터링으로 **즉각적인 문제 대응**
- 추후 사이트 추가 시 **같은 구조 재활용** (확장 비용 절감)

## 2. 시스템 아키텍처

```
사용자(클라이언트/개발자)
        ↓
[Vercel]
 ├─ 클라이언트 대시보드 (Next.js)
 └─ 관리자 대시보드 (Next.js)
        ↓ HTTPS API
[Cloudflare Workers]         ← 인증·API·비즈니스 로직
        ↓
[Cloudflare D1 (SQLite)]     ← 계정·설정·작업 이력 DB
[Cloudflare R2]              ← 세션 쿠키·상세 로그 파일
        ↓ 실행 명령
[Cloudflare Containers]      ← Playwright 10개 계정 병렬 실행
        ↓ 사이트 API 호출
[allin42.com]
```

### 각 구성요소 역할

| 구성 요소 | 역할 | 비고 |
|---------|------|------|
| Vercel | 대시보드 프론트엔드 호스팅 | 무료 서브도메인, 자동 SSL |
| Cloudflare Workers | API 서버 | 서버리스, 글로벌 CDN |
| Cloudflare D1 | 관계형 DB | 계정·설정·로그 메타데이터 |
| Cloudflare R2 | 오브젝트 스토리지 | 세션 쿠키·대용량 로그 |
| Cloudflare Containers | 매크로 실행 엔진 | Playwright 24/7 상주 |
| Claude API (Haiku 4.5) | AI 댓글 생성 | 한국어 품질 우수, 저비용 |

## 3. 기술 스택 선택 근거

### Vercel (대시보드 호스팅)

- **이유**: Next.js 네이티브 지원, 배포 자동화, 무료 티어로 충분
- **장점**: `*.vercel.app` 서브도메인 무료, SSL 자동, 글로벌 CDN
- **대안 대비**: Cloudflare Pages도 가능하나 Next.js DX는 Vercel 우위

### Cloudflare Workers + D1 + R2

- **이유**: 통합 플랫폼, 무료 티어 관대, 비용 예측 용이
- **장점**: R2는 외부 전송 비용 무료(S3 대비 월 수만원 절감), D1은 Edge 분산 SQL
- **대안 대비**: AWS Lambda+DynamoDB+S3 조합 대비 50~70% 저렴

### Cloudflare Containers (매크로 실행)

- **이유**: Playwright 24/7 상주 실행 가능, 컨테이너 기반이라 확장 용이
- **장점**: GitHub Actions 6시간 제약 완전 제거, 10계정 병렬 네이티브 지원
- **대안 대비**:
  - GitHub Actions: 6시간 제한·무료 한도 초과 시 비용 예측 어려움
  - Browser Rendering: 버스트 과금이라 24/7 운영 시 과도 비용
  - 별도 VPS: 관리 부담·Cloudflare 생태계 이탈

### Claude API (claude-haiku-4-5)

- **이유**: 빠른 응답(~1초), 저렴한 토큰 단가, 한국어 품질 우수
- **장점**: 댓글 1건당 약 $0.0003 수준
- **대안 대비**: GPT-4o-mini 대비 한국어 자연스러움·뉘앙스 우위

## 6. 10계정 병렬 운영 설계

### 실행 구조

```
Cloudflare Container
 ├─ Worker 1: 계정 A → Playwright 세션 유지 → 댓글 작성 루프
 ├─ Worker 2: 계정 B → Playwright 세션 유지 → 댓글 작성 루프
 ├─ ... (동일 구조)
 └─ Worker 10: 계정 J → Playwright 세션 유지 → 댓글 작성 루프
```

### 병렬 실행 이점

| 항목 | 단일 계정 | 10계정 병렬 |
|------|----------|-----------|
| 시간당 작성량 | 약 400건 | **약 4,000건** |
| 하루(10시간 운영) | 약 4,000건 | **약 40,000건** |

> 실제 작성량은 사이트 측 하루 한도에 따라 제한됩니다. 계정별 안전 한도 설정으로 어뷰징 방지.

### 공유 IP 관련 주의사항

- 10개 계정이 모두 같은 Cloudflare IP에서 접근 시 사이트 측 어뷰징 감지 가능성
- **대응책 1**: 계정별 요청 간격 랜덤화 (User-Agent·Timing 변화)
- **대응책 2 (선택)**: Cloudflare WARP·프록시 IP 분산 (+추가 비용 발생 시 협의)

## 7. 자동 로그인 동작 방식

### 기본 흐름

```
[세션 쿠키 보유] → 사이트 접근 → 정상 동작
      ↓ 세션 만료 감지 (401 또는 로그인 페이지 리디렉션)
[자동 재로그인 트리거]
      ↓
[R2에서 계정정보 복호화] → Playwright 로그인 → 새 세션 저장
      ↓
[원래 작업 재개]
```

### 특징

- **사용자 개입 불필요**: 로그아웃되어도 무감각하게 계속 운영
- **세션 쿠키는 R2 암호화 저장** (AES-256)
- **재로그인 실패 3회 시** → 관리자 대시보드 알림
- 캡차 없음 확인됨 → 추가 모듈 불필요

## 8. 보안 설계

- 사이트 비밀번호 **AES-256 암호화** 후 D1에 저장
- 암호화 키는 Cloudflare Secret Manager 관리 (코드·DB 분리)
- 대시보드 **HTTPS 강제** (Vercel·Cloudflare 자동)
- 로그인 **이메일 + 비밀번호 + 2FA (선택)**
- 관리자 대시보드 **접근 IP 제한** 가능
- 모든 API 호출 **JWT 인증**
- 세션 토큰 **24시간 만료**
- 감사 로그(audit log) 자동 기록

## 16. v1(설치형) vs v2(관리형 SaaS) 전체 비교표

### ① 실행·운영 환경

| 항목 | v1 (설치형) | v2 (관리형 SaaS) |
