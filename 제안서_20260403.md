## 1. 배경 및 목표

### 현재 상황

- allin42.com 자유게시판에 수동으로 댓글을 작성 중
- 하루 약 8시간 분량의 반복 작업 발생
- 댓글 간 9초 쿨타임 규정으로 인해 자동화 없이는 지속 불가

### 구현 목표

- 게시글 내용을 분석해 자연스러운 한국어 댓글을 AI가 생성
- 9초 쿨타임을 준수하며 자동 등록
- **세션 만료 시 자동 재로그인** (사이트가 강제 로그아웃시켜도 무중단 운영)
- GitHub Actions로 별도 서버 없이 무료 운영

### 기대 효과

- 댓글 작업 시간 100% 자동화
- 월 운영비 최소화 (API 호출 비용만 발생)
- 24시간 모니터링 없이 스케줄 기반 자동 실행
- **로그아웃으로 인한 다운타임 제거** (외출·수면 중에도 연속 작동)

## 3. 기술 스택 선택 근거

### GitHub Actions

- **이유**: 별도 VPS/서버 없이 클라우드 실행 가능, 무료 플랜 활용
- **장점**: 코드 관리 + 실행 환경이 하나로 통합, 수동 트리거 지원
- **한계**: Job당 최대 6시간 → 8시간 운영 시 분할 실행 필요

### Playwright (Python)

- **이유**: 사이트가 Supabase 세션 인증(브라우저 쿠키 기반)을 사용하므로 headless 브라우저 필수
- **장점**: 로그인 후 fetch() 그대로 재사용 가능, JavaScript 실행 환경 내장
- **대안 대비**: requests/httpx 단독으로는 Supabase 세션 처리 불가

### Claude API (claude-haiku-4-5)

- **이유**: 빠른 응답 속도, 저렴한 토큰 단가, 한국어 품질 우수
- **장점**: Haiku 모델 사용 시 댓글 1건당 약 $0.0002 수준
- **대안**: GPT-4o-mini도 유사 비용이나 Claude 한국어 자연스러움 우위

## 6. 주요 코드 구조 (예시)

### 댓글 API 호출 (브라우저 fetch 방식)

```python
async def post_comment(page, board_id, text):
    payload = {
        "action": "CommentWrite",
        "requested_at": datetime.now(timezone.utc).isoformat(),
        "comment_board_id": board_id,
        "comment_contents": f"<p>{text}</p>",
        "comment_editor_state": make_lexical_state(text),
        "comment_status": "y"
    }
    result = await page.evaluate("""async (payload) => {
        const resp = await fetch('/api/board/detail', {
            method: 'POST',
            headers: {'Content-Type': 'application/json'},
            body: JSON.stringify(payload)
        });
        return await resp.json();
    }""", payload)
    return result
```

### AI 댓글 생성 (Claude Haiku)

```python
def generate_comment(post):
    client = anthropic.Anthropic()
    msg = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=150,
        messages=[{"role": "user", "content":
            f"글 제목: {post['title']}\n내용: {post['contents']}\n"
            f"자연스러운 한국어 댓글 1~2문장 작성"
        }]
    )
    return msg.content[0].text.strip()
```

## 9. 자동 로그인 설계

### 동작 흐름

```
[정상 작업 중] → 사이트 API 호출
      ↓ 401/403 응답 또는 로그인 페이지 감지
[세션 만료 인식]
      ↓
[저장된 계정정보로 Playwright 자동 재로그인]
      ↓ 새 세션 쿠키 확보
[원래 작업 재개] → 중단 없이 계속 진행
```

### 실제 사용 시나리오

> **예시**: 매크로를 켜놓고 **외출 중 사이트가 강제 로그아웃을 발생**시킨 경우 — 기존 방식에서는 복귀할 때까지 **수 시간 동안 작업이 정지된 상태**로 남아 그 시간만큼 손실이 발생합니다. 본 프로그램은 로그아웃을 **즉시 감지하여 자동으로 재로그인**한 뒤 작업을 이어가므로, 사용자 개입 없이 **24시간 연속 운영**됩니다.

### 특징

- **세션 만료 감지**: HTTP 응답 코드 + 리디렉션 URL 기반 자동 판별
- **재로그인 간격 제어**: 사이트 부하 방지를 위해 재시도 간격 적용 (무한 루프 방지)
- **실패 시 로그 기록**: 재로그인 3회 연속 실패 시 실행 로그에 명확히 표시 (수동 확인용)
- **자격증명 보관**: GitHub Actions Secrets에 암호화 저장, 코드·로그에 노출되지 않음
